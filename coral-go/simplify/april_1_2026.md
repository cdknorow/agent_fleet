# Codebase Simplification Review — April 1, 2026

Full-team review of the Coral Go codebase for dead code, redundancy, bugs, and simplification opportunities. Six agents reviewed all packages, frontend assets, tests, CLI commands, and documentation.

## Critical Bugs

These are functional defects that affect correctness or data integrity.

### BUG-1: Missing SELECT/INSERT columns cause silent data loss
- **Files:** `internal/store/sessions.go:755-905`
- **Issue:** `GetAllLiveSessions`, `GetBoardSessions`, and `GetLiveSession` omit `backend` and `pid` from SELECT. Sessions always wake as tmux even if registered as PTY. `ReplaceLiveSession` INSERT silently drops `is_job`, `backend`, `capabilities`, `model`, `tools`, `mcp_servers` — 6 columns lost on every replace.
- **Fix:** Add missing columns to SELECT and INSERT statements.
- **Severity:** Critical

### BUG-2: TOCTOU race in scheduler concurrency limit
- **File:** `internal/background/scheduler.go:248-273`
- **Issue:** Lock released before goroutine registers in `s.running`, so multiple jobs can bypass `maxConcurrent`.
- **Fix:** Pre-register placeholder before releasing lock.
- **Severity:** Critical

### BUG-3: Stale PTY session map after server restart
- **File:** `internal/ptymanager/tmux_backend.go`
- **Issue:** `TmuxBackend.sessions` map is a stale local cache never re-hydrated after server restart. All methods fail silently on restart.
- **Fix:** Remove map and use live tmux lookups, or add re-hydration on startup.
- **Severity:** Critical

### BUG-4: `--no-browser` flag not forwarded in coral-tray
- **File:** `cmd/coral-tray/main.go:59, 116`
- **Issue:** Parent exits after spawning child, and child opens the browser anyway because `--no-browser` is not passed to the re-spawn args.
- **Fix:** Add `if *noBrowser { args = append(args, "--no-browser") }` to the re-spawn args.
- **Severity:** High

### BUG-5: `persistResults` called with cancelled context
- **File:** `internal/background/workflow_runner.go:247`
- **Issue:** After context cancellation, `persistResults` is called with the already-cancelled context and silently fails.
- **Fix:** Use `context.Background()` for the persist call.
- **Severity:** High

### BUG-6: `DeleteProject` ignores transaction errors
- **File:** `internal/board/store.go:869-871`
- **Issue:** All three `tx.ExecContext` errors in the transaction are ignored.
- **Fix:** Capture and return errors.
- **Severity:** High

### BUG-7: Diverged prompt constants between system.go and agent.go
- **File:** `internal/server/routes/system.go:114-128` vs `internal/agent/agent.go`
- **Issue:** `DefaultOrchestratorSystemPrompt` etc. are duplicated and have diverged in content. Frontend shows one set of defaults, agent runs with different ones.
- **Fix:** Reference `agent.` package constants, delete local copies in system.go.
- **Severity:** High

### BUG-8: `getLogStatus` opens file then ignores it
- **File:** `internal/server/routes/sessions.go:181-193`
- **Issue:** Opens file with `os.Open` + `f.Seek`, then re-reads from scratch via `os.ReadFile`. The open/seek is entirely wasted.
- **Fix:** Remove `os.Open`/`f.Seek`, keep `os.ReadFile` with start-offset slice.
- **Severity:** Medium

### BUG-9: `ImportTheme` silently swallows write errors
- **File:** `internal/server/routes/themes.go:433`
- **Issue:** `os.WriteFile` error discarded, always returns `ok:true`.
- **Fix:** Check error, return 500.
- **Severity:** Medium

### BUG-10: `GetSessionDetail` returns 200 for not-found
- **File:** `internal/server/routes/history.go:406-410`
- **Issue:** Missing session returns HTTP 200 with error body instead of 404.
- **Fix:** Use `errNotFound`.
- **Severity:** Medium

### BUG-11: `ReadMessages` omits `target_group_id`
- **File:** `internal/board/store.go:394-401`
- **Issue:** SELECT omits `target_group_id` — always nil in results.
- **Fix:** Add to SELECT.
- **Severity:** Medium

### BUG-12: `persistResults` called twice on failure path
- **File:** `internal/background/workflow_runner.go:310-319`
- **Issue:** On `!ContinueOnFailure` path, first call at line 312 is immediately overwritten by line 319.
- **Fix:** Remove redundant first call.
- **Severity:** Medium

### BUG-13: `UpdateWorkflow` silently ignores JSON unmarshal errors
- **File:** `internal/server/routes/workflow.go:231, 243, 270`
- **Issue:** `json.Unmarshal` errors silently ignored, producing misleading error messages.
- **Fix:** Check errors, return `errBadRequest`.
- **Severity:** Medium

### BUG-14: `time.Sleep(5s)` blocks HTTP handler in KillRun
- **File:** `internal/background/workflow_runner.go:144-152`
- **Issue:** SIGTERM + 5s grace + SIGKILL blocks the HTTP handler goroutine.
- **Fix:** Move to background goroutine.
- **Severity:** Medium

### BUG-15: `copyDir` silently drops write errors
- **File:** `internal/agent/claude.go:298-316`
- **Issue:** `os.WriteFile` errors are not returned or logged.
- **Fix:** Add error propagation or logging.
- **Severity:** Medium

### BUG-16: `CoralAPI` discards `io.ReadAll` error
- **File:** `internal/hooks/hooks.go:84`
- **Issue:** Caller gets malformed JSON on read failure.
- **Fix:** Return the error.
- **Severity:** Medium
